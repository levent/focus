# Focus

**Focus** is a lightweight daily productivity app built with Ruby on Rails. Every day it guides you through a simple two-part workflow:

- **Morning** – Set your primary focus for the day and note any anticipated blockers.
- **Evening** – Reflect on whether you achieved your goal, and capture a reason if you didn't.

The app automatically detects where you are in the day's cycle and shows the right prompt, so you never have to navigate anywhere – just open it and answer.

## Features

- 📝 **Morning check-in** – capture your primary focus and optional blockers.
- 🌙 **Evening review** – mark the day achieved or log the reason it wasn't.
- 📋 **Daily summary** – a read-only view of the full day once the review is saved.
- 🔄 **State-aware routing** – one URL, three smart views (morning / evening / summary).
- 💅 **Clean, minimal UI** – Tailwind CSS with a calm stone/emerald/red palette.
- 🚀 **Production-ready** – Docker, Kamal deployment, Solid Queue/Cache, Thruster.

## Prerequisites

- **Ruby** 3.4.8 (use [mise](https://mise.jdx.dev/) or [rbenv](https://github.com/rbenv/rbenv))
- **Bundler** 2+ (`gem install bundler`)
- **Node.js** – only needed if you modify JavaScript imports
- **SQLite 3** development libraries

```bash
# macOS
brew install sqlite3

# Debian / Ubuntu
sudo apt-get install libsqlite3-dev
```

## Getting Started

```bash
# 1. Clone the repository
git clone https://github.com/levent/focus.git
cd focus

# 2. Install the correct Ruby version (with mise)
mise install

# 3. Install gem dependencies
bundle install

# 4. Set up the database
bin/rails db:create db:migrate

# 5. Start the development server (Rails + Tailwind watcher)
bin/dev
```

The app will be available at **http://localhost:3000**.

### Environment Variables

For production you will need a `RAILS_MASTER_KEY` (generated by Rails credentials):

```bash
bin/rails credentials:edit   # creates config/master.key if it doesn't exist
```

Never commit `config/master.key` – it is already listed in `.gitignore`.

## Running Tests

The project uses **RSpec** for unit, request, and full-stack system tests (headless Chrome via Cuprite).

```bash
# Run the full test suite
bundle exec rspec

# Run only model tests
bundle exec rspec spec/models

# Run only system tests
bundle exec rspec spec/system

# Run only request tests
bundle exec rspec spec/requests
```

### Linting & Security

```bash
# RuboCop (style checks)
bundle exec rubocop

# Brakeman (Rails security scanner)
bundle exec brakeman

# Bundler Audit (known gem vulnerabilities)
bundle exec bundler-audit check --update
```

All of the above run automatically in CI via GitHub Actions on every push and pull request.

## Deployment

Focus ships with [Kamal](https://kamal-deploy.org/) for zero-downtime Docker deployments.

### 1. Build and push the Docker image

```bash
kamal build push
```

### 2. Configure your server

Edit `config/deploy.yml` and fill in:

```yaml
service: focus
image: your-registry/focus

servers:
  web:
    - your-server-ip

registry:
  server: your-registry
  username: your-username
  password:
    - KAMAL_REGISTRY_PASSWORD   # stored in .kamal/secrets

env:
  secret:
    - RAILS_MASTER_KEY
```

Populate `.kamal/secrets` with your actual credentials (never commit this file).

### 3. Deploy

```bash
# First deploy
kamal setup

# Subsequent deploys
kamal deploy
```

### Manual / plain Docker

```bash
docker build -t focus .
docker run -e RAILS_MASTER_KEY=<your-key> -p 80:80 focus
```

## Contributing

Contributions are welcome! Here is the recommended workflow:

1. **Fork** the repository and create a feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. **Make your changes** – keep them focused and atomic.
3. **Add or update tests** – all new behaviour should be covered by RSpec specs.
4. **Run the full suite** to confirm nothing is broken:
   ```bash
   bundle exec rspec && bundle exec rubocop
   ```
5. **Open a pull request** against `main` with a clear description of what you changed and why.

Please follow the existing code style (enforced by [RuboCop Rails Omakase](https://github.com/rails/rubocop-rails-omakase)) and keep pull requests small and focused.
